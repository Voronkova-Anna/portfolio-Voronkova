# Проект: Разработка спецификации требований для платформы CodeUnity

## Цель проекта:
 - Создать веб-платформу для эффективной коммуникации IT-специалистов, обеспечивающую поиск команд и участников в проекты, а также обмен знаниями через раздел Q&A. 

## Выполненные задачи: 
- Собраны и структурированы функциональные и нефункциональные требования для MVP платформы(98 требований, включая авторизацию через OAuth, ролевую модель и бизнес-правила);
- Разработана диаграмма вариантов использоавния(Use Case), отражающая иерархию пользователей и их взаимодействие с системой.
- Детально описаны 60+ вариантов использования для разделов "Профиль", "Проекты", "О нас", "Административная панель" с основными и альтернативными сценариями;
- Составлена физическая модель данных (ERD) и словарь данных для 11 ключевых сущностей: пользователи, профили, проекты, отклики, приглашение участников, публикация вопросов, лайки, сообщения в поддержку, блокировка контента);
- Разработаны 13 диаграмм последовательности в нотации UML, демонстрирующих взаимодействие фронтенда и бэкенда (регистрация, создание проекта, отклик, приглашение, публикация вопроса, лайки, сообщение в поддержку, блокировка контента);
- Описана логика работы Frontend (SPF на HTML/CSS/JS) и Backend (Django REST API), а также API-контракты в Swagger;
- Поставлены задачи на разработку ключевых операций API.

## Результат:
- Подготовлена полная спецификация веб-платформы CodeUnity, включающая функциональные и нефункциональные требования, модели данных и сценарии взаимодействия;
- Проработаны архитектурные и интеграционные аспекты, обеспечивающие совместимость Django REST API и PostgreSQL;
- Результаты готовы к реализации командой разработчиков и дальнейшей интеграции в продукт, составлены задачи на разработку API.
